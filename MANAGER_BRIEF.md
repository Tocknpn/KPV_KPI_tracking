# SalesTrack Pro — Manager Brief

*A quick read for anyone deciding whether/how this app fits into how the business runs.*

---

## What Is It?

SalesTrack Pro is a desktop app that tracks daily sales performance for gold & jewelry staff across all branches, turns that into a KPI score automatically, and shows everyone — from a single sales rep up to top management — exactly where they stand against their monthly target, updated daily.

It replaces manual spreadsheet tracking and end-of-month guesswork with a live, auto-scored system everyone can check anytime.

---

## How It Works (in plain terms)

1. **Each branch's Accountant Officer uploads the day's sales** (jewelry weight, bar weight, quantity sold per rep) via a simple file upload — no manual data entry screen, just drop a file.
2. **The app scores it automatically** the moment it's uploaded — every rep's jewelry, bar, and quantity numbers convert into points using rates set by HR (e.g. weight × rate, or quantity tier multipliers), then compared against that rep's monthly target.
3. **Everyone sees their own slice** — a rep sees their own number, a supervisor sees their team, a branch manager sees their branch, and management sees everything — automatically, no one has to "send" anyone a report.
4. **All data syncs to a shared Google Sheet** in the background, so every device — head office, every branch — is always reading the same numbers, even offline-first on each machine.
5. **A built-in approval safety net** stops accidental double-uploads or overwrites of already-recorded sales — a correction has to go through an Accountant Manager, never silently overwrites real numbers.

---

## Daily Routine, by Level

| Level | What they do, day to day |
|---|---|
| **Accountant Officer** (per branch) | Uploads that day's sales file. That's the entire daily job. |
| **Sales Supervisor** | Checks their team's KPI % and progress projection each morning — sees who's behind, no manual calculation needed. |
| **Branch Manager** | Same view as a supervisor, but across every team in their branch — compares teams against each other. |
| **Accountant Manager** | Reviews upload activity across all branches; the only one who can approve/clear a correction if an officer needs to re-upload. |
| **HR** | Manages the roster (who works where) and the KPI rates/targets — monthly, not daily, but the one place rates ever get adjusted. |
| **Top Manager / Admin** | Logs in periodically for a company-wide read — no data entry, just oversight. Admin separately manages user accounts and access. |

Nobody has to manually compile a report for anyone else — the dashboard already shows the current, live number for whatever level you're looking from.

---

## Why This Beats the Old Way

- **No more end-of-month surprises** — every rep's KPI % updates the day the sale gets uploaded, not weeks later.
- **One source of truth** — every branch and every manager reads the same live numbers, no separate spreadsheets drifting out of sync with each other.
- **Built-in accountability** — every upload, every correction, every user action is logged (Audit Log) — nobody can quietly change a number without a trace.
- **Each role only sees and touches what's theirs** — a supervisor can't accidentally edit another branch's data; an officer can't overwrite a number that's already locked in.
- **Works without constant internet** — each device keeps its own local copy and syncs in the background, so a branch with patchy internet isn't blocked from working.
- **No spreadsheet skills needed** — the scoring math (rates, tiers, targets) lives in one place (KPI Settings), set once by HR, applied automatically to every upload from then on.
- **Scales without extra admin work** — adding a new branch, supervisor, or rep is a roster entry, not a new spreadsheet to build and maintain.
