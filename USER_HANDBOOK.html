1. Logging In & Connecting

Everyone

Every screen starts here. Enter your username and password, click Sign In.

After you click Sign In, the app pulls the latest data from Google Sheets before letting you in, so you never see stale numbers:

Tip If there's no internet, the app still lets you in using whatever was last saved on this device — it just shows a small notice instead of blocking you.

Switching the connected database (Test vs Production)

A small icon in the top-right corner of the Login screen lets you point this device at a different Google Sheet entirely.

Warning Switching wipes this device's local data first, then pulls everything fresh from the new Sheet. Don't do this casually on a device that's actively being used — anything not yet synced is lost.

2. Dashboard

AdminHR Acc. ManagerSales Sup Branch MgrTop Mgr

Your landing page after login (for roles that have it). A live snapshot of the current month — total Jewelry/Bar/Quantity, overall KPI score, and your Top 10 performers.

  • Use the branch and month/date filters top-right to change what you're viewing.
  • The KPI Score ring shows your overall % against target for the selected period.

Tip A supervisor only ever sees their own team here; a branch manager sees their whole branch; admin/HR/top manager see everything.

3. Daily Entry Upload

Accountant OfficerAccountant Manager

This is how daily sales actually get into the system — by uploading a spreadsheet file, not typing into a form. An Accountant Officer uploads for their own branch only; an Accountant Manager can upload for any branch.

  1. Click Download Template — gives you an .xlsx pre-filled with this month's reps for your branch.
  2. Fill in Date, Jewelry (Baht), Bar (Baht), Qty for each rep. Leave everything else as-is.
  3. Drag the file onto the drop zone (or click to browse), then click Import Daily Data.
Note If a row already has a record for that rep + date, it gets rejected to protect existing numbers from being silently overwritten. If that happens, ask an Accountant Manager to clear the conflicting batch first (see Upload History below), then re-upload.

4. Upload History

AdminHRAcc. OfficerAcc. ManagerTop Mgr

Where you check that every branch is actually uploading on schedule, and where an Accountant Manager fixes a bad upload.

  • Branch Coverage Matrix — at a glance, which branches have entered today's data and which haven't.
  • Sales Upload Records — Approval (Accountant Manager only) — find a bad batch, click Delete & Allow Resubmit to clear it so the officer can re-upload corrected data.
  • Upload Log at the bottom lists every file ever uploaded, who uploaded it, and its status.
Note Deleting a batch is irreversible inside the app — confirm with the Accountant Officer before clearing, not after.

5. Roster — Reps

HRHR SupportTop Mgr

The org-chart job — who works where, for which branch, under which supervisor. The roster is month-aware: editing this month never changes how last month's report reads.

  • Add Rep — manually add one person.
  • Upload Roster — bulk-upload a spreadsheet of many reps at once.
  • The month picker (top-right) controls which month you're viewing/editing — pick the right one before making backdated changes.
  • Check Show Inactive to reveal deactivated reps — only then do Reactivate and permanent Delete buttons appear.
Tip Bulk roster files need an Effective_Date column — that's what decides which month the change counts from, there's no separate month dropdown for the upload itself.

9. KPI Settings

Admin (Defaults)HR (Monthly)Top Mgr

This is the one screen that controls how every score gets calculated — but Admin and HR see two different jobs on it.

Admin view — standing Defaults

Admin sets the standing values that any month without its own override falls back to. No month picker — Defaults apply forever until changed again.

HR view — this month's settings

HR picks a month, optionally clicks Use Defaults to pull Admin's standing values in as a starting point, adjusts anything different this month, then clicks Save All to confirm.

Important HR must click Save All every month — even if nothing changed — to mark that month "Confirmed." Skipping it leaves a warning banner up on Dashboard/KPI Report (numbers still calculate fine using the last known values, but the banner stays until explicitly confirmed).

10. Settings & Users

AdminHR (Connection only)

Connection Settings

Keeps the Google Sheets connection healthy — manual Push/Pull if the "unsynced" count isn't clearing on its own.

User Management

Create/deactivate accounts, assign roles and branches, override an individual person's menu access beyond their role's default.

Tip Each save auto-pushes to the Google Sheet immediately. Forgot someone's password? Either type a new one in their edit form, or look it up directly on the Users sheet tab (stored in plain text there by deliberate design).

AdminHR

A record of every meaningful action any user took — logins, roster changes, permission edits, KPI confirms — who did it, when, and what it affected.
